# Описание проекта
Это легковесный веб-трекер для отслеживания конкретного авиарейса (LY 88). 
Проект размещается на GitHub Pages. Главный приоритет — скорость работы, простота и отсутствие сложных систем сборки.

# Технологический стек
- **HTML:** HTML5 (семантическая разметка).
- **CSS:** Tailwind CSS (подключен через CDN). Никакого кастомного CSS в файле `style.css`, если задачу можно решить утилитарными классами Tailwind.
- **JavaScript:** Vanilla JS (ES6+). Никаких фреймворков вроде React или Vue. Никакого TypeScript, только чистый JavaScript.

# Правила написания кода (JavaScript)
1. Используй современные стандарты: `const` / `let`, стрелочные функции, деструктуризацию.
2. Для асинхронных запросов используй `async/await` и `fetch`. Всегда оборачивай запросы в `try/catch` для обработки ошибок.
3. Названия переменных и функций должны быть на английском языке, понятными и говорящими (например, `fetchWeather`, а не `getW`).
4. Не удаляй существующие комментарии в коде и добавляй краткие комментарии к новым функциям.

# Интеграции и API
1. По возможности используй бесплатные публичные API, которые не требуют авторизации (CORS-friendly, HTTPS).
2. Никогда не хардкодь приватные API-ключи в файлы. Если ключ нужен, пиши инструкцию, как его добавить.

# Git и развертывание
1. Коммиты должны быть на английском языке, краткими и отражать суть изменений (например, "Add weather fetch logic").
2. Помни, что проект работает на GitHub Pages. Все пути к файлам (картинки, скрипты) должны быть относительными.
